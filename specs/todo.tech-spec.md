# Tech Spec: 심플 투두리스트 (Quick Spec)

## 배경/목표
`specs/todo.md` 요구사항을 기준으로, 단일 파일 `src/index.html`에서 동작하는 심플 투두리스트를 구현한다.

## 범위
- **포함**
  - 할 일 입력 후 **추가**
  - 할 일 항목별 **삭제**
  - **깔끔/모던 UI**
  - **HTML/CSS/JavaScript만 사용**
  - **단일 파일**(`index.html`)로 동작
- **제외(요구사항 외)**
  - 완료 체크/필터/정렬 *(개선 항목으로 선택 구현 가능)*
  - 로컬스토리지 저장 *(개선 항목으로 선택 구현 가능)*
  - 서버 연동

## UI/디자인 요구사항
- 기본 배경은 **흰색 계열**로 깔끔하게 구성
- 버튼은 시각적으로 명확한 **Primary 스타일**
- (이 저장소의 `.cursorrules`가 적용되는 경우)  
  - 모든 `<button>`은 `class="ncds-btn-primary"` 사용
  - 모든 `<input>`은 `class="ncds-input"` 사용
  - `ncds-btn-primary` 배경색은 **NHN Red `#e61e2b`**

## 데이터 모델(메모리)
- `todos: Array<{ id: string, text: string }>`
- `id`는 삭제 대상 식별을 위해 유니크해야 함

## 동작 스펙

### 기능 1: 할 일 추가
- 사용자가 입력창에 텍스트 입력
- **[추가]** 버튼 클릭(또는 Enter 제출) 시 목록에 항목 추가
- 입력은 `trim()` 처리 후 빈 문자열이면 무시
- 추가 후 입력창은 비우고 포커스 유지(UX)

### 기능 2: 할 일 삭제
- 각 항목 우측에 **[삭제]** 버튼 존재
- 해당 버튼 클릭 시 **그 항목만** 목록에서 제거

### 기능 3: 빈 상태/카운트(권장)
- 목록이 비어있으면 “아직 할 일이 없어요…” 같은 안내 문구 표시
- 총 개수 표시(예: “총 N개”)

## 스토리(작업 단위) & 수용 기준(AC)

### Story 1 — 투두 추가
- **AC1**: 입력 후 [추가]를 누르면 목록에 새 항목이 보인다.
- **AC2**: 공백만 입력 후 [추가]를 누르면 아무 일도 일어나지 않는다.
- **AC3**: 추가 후 입력창이 비워지고, 다시 입력할 수 있도록 포커스가 유지된다.

### Story 2 — 투두 삭제
- **AC1**: 각 항목 옆에 [삭제] 버튼이 있다.
- **AC2**: [삭제] 클릭 시 해당 항목이 목록에서 사라진다.

### Story 3 — UI(화이트 + 모던)
- **AC1**: 전체 배경은 흰색/밝은 톤이고 여백/정렬이 깔끔하다.
- **AC2**: Primary 버튼이 명확하고(색상 대비), hover/active 등 기본 인터랙션이 어색하지 않다.
- **AC3**: (해당 시) `.cursorrules`의 NCDS 클래스 규칙을 위반하지 않는다.

## 개선 스토리(선택)

### Improve 1 — 중복 입력 방지
- **AC1**: 이미 존재하는 할 일을 동일한 내용으로 다시 추가할 수 없다. (대소문자/공백 차이는 동일로 처리)
- **AC2**: 중복 입력 시 사용자에게 힌트/피드백이 제공된다.

### Improve 2 — 완료 체크 + 완료 스타일
- **AC1**: 각 항목에 완료 체크(체크박스)가 존재한다.
- **AC2**: 체크 시 완료 스타일(취소선/색상 변화)이 적용된다.
- **AC3**: 완료 상태는 저장될 수 있다(선택).

### Improve 3 — 일괄 삭제
- **AC1**: “완료만 삭제”로 완료 항목만 제거할 수 있다.
- **AC2**: “전체 삭제”로 모든 항목을 제거할 수 있다.

### Improve 4 — 필터(전체/진행중/완료)
- **AC1**: 전체/진행중/완료 필터 버튼으로 표시 목록이 바뀐다.
- **AC2**: 필터가 적용되어도 총 개수(전체)는 유지 표시된다.

### Improve 5 — 드래그 정렬
- **AC1**: 사용자가 항목을 드래그해 순서를 변경할 수 있다.
- **AC2**: (권장) 정렬 결과가 저장되어 새로고침 후에도 유지된다.

## 테스트 플랜(수동)
- **추가**: “운동 30분” 입력 → [추가] → 목록에 표시되는지 확인
- **빈 입력**: “   ” 입력 → [추가] → 추가되지 않는지 확인
- **중복**: 동일 문구를 다시 입력 → 중복 추가되지 않는지 확인
- **삭제**: 임의 항목 [삭제] → 해당 항목만 제거되는지 확인
- **완료**: 체크박스 체크/해제 → 스타일 토글 확인
- **필터**: 전체/진행중/완료 버튼 → 목록이 올바르게 바뀌는지 확인
- **일괄 삭제**: 완료만 삭제/전체 삭제 버튼 동작 확인
- **정렬**: 항목 드래그 → 순서 변경 + (저장 시) 새로고침 후 유지 확인
- **UI**: 모바일 폭(좁은 화면)에서도 입력/버튼이 깨지지 않는지 확인
- **(개선) 저장**: 항목 추가 후 새로고침 → 목록이 유지되는지 확인
